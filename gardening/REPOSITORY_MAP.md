# 🗺️ Repository Map — What Everything Is & Why It's Here

This document explains the entire Greenshadow Garden repository: what we track, why we track it, and where to find what you need.

---

## What Is This Repository?

**Greenshadow Garden is a complete, living gardening knowledge base for one property in Fuquay-Varina, NC (Zone 7b).**

It contains:
- **Detailed plans** for 7 major garden beds + 4 small gardens (11 gardens total)
- **Inventory tracking** for 228 seeds and plants with sources and assignments
- **Growing guides** for what to plant when in this specific zone
- **Maintenance reference** for care, troubleshooting, soil, pests, diseases
- **Herbology reference** — healing uses for 25+ herbs on the property
- **Calendar** synced to your phone with all planting/maintenance dates
- **Historical archives** of planning explorations and drafts

**This is not decorative.** Every decision is tracked. Every plant has a purpose. This is a working system that improves itself each year.

**What each file contains:**
- Garden name, location, size, and purpose
- Color scheme or theme (if applicable)
- Full plant list with quantities and spacing
- Layout diagram or grid coordinates
- Planting timeline (when to plant what)
- Care notes (watering, weeding, harvesting, etc.)
- Companion planting notes
- Notes from previous owner or design thinking

**Special case — Potion Patch (medicinal healing circle):**
Each major garden has one file, but Potion Patch has multiple detailed guides due to its complexity:
- **Potion_Patch.md** — Hub file with overview + links
- **Potion_Patch_Concentric_Design.md** — Full design reasoning + every plant explained
- **Potion_Patch_Positioning_Quick_Guide.md** — Practical measurements + placement
- **Potion_Patch_Planting_Guide.md** — 4-phase timeline + step-by-step instructions
- **Potion_Patch_Shopping_List.md** — Plants, seeds, costs, budgets
- **Potion_Patch_Visual_Guide.md** — Diagrams, spacing, growth timeline
- **Potion_Patch_Accessories_Guide.md** — Mirror, bee house, stepping stones, etc.

**How to use:** For other gardens, pick one file and follow the plan. For Potion Patch, start with Potion_Patch.md, then dive into the detailed guides as needed.

### **inventory/** — Tracking What You Own (5 Files)

Complete records of everything in the garden and assignments to beds.

| File | Purpose | Format |
|------|---------|--------|
| **Master_Seed_and_Plant_List.md** | All 228 seeds and plants — suppliers, varieties, garden assignments, notes | Markdown table |

**What these files contain:**
- Complete source information (seed company, catalog page, price)
- Plant type (seed, transplant, perennial, annual)
- Garden assignment (where it should go)
- Planting dates (when to start seed, when to transplant)
- Quantity (how many seeds or plants you have)
- Notes (special care, success rate, design thinking)

**How to use:**
- Want to know what you own? → Master_Seed_and_Plant_List.md
- Planning Bloomveil layout? → Bloomveil_Grid.csv
- Don't know what to do with something? → Inventory_Audit_March_2026.md

### **archive/** — Historical Explorations (53 Files)

Old drafts, superseded plans, and experimental ideas from the planning process. **You don't need these to run the garden.**

**What's in here:**
- **Old bed guides** that were consolidated into the clean versions (e.g., archive/Bloomveil.md vs. beds/Bloomveil.md)
- **Abandoned ideas** (Butterfly Garden, Children's Garden, Bird Feeder Plants — never implemented)
- **Old inventory systems** (Seeds_Inventory_Early_2025.md, Plants_List.md before consolidation)
- **Experimental plans** (various grid iterations, alternatives)
- **One-off planning docs** (Saturday_Gardening_Plan, Sunday_March_9_Todo)
- **Idea captures** (Garden_Style_Mood_Boards, social media hashtags, design explorations)

**Why keep them?**
- They contain real planning history and decision-making
- They show why current plans exist
- Some have useful details (Garden_Photography_Guide, Garden_Arbor_DIY)
- They preserve the evolution of the garden concept

**When to look here:**
- You're curious about how decisions were made
- You want context for current plans
- You're looking for an old idea to resurrect
- Almost never for active gardening

## 🔍 How to Find What You Need

### "I want to plant [thing]"
1. Check [Planting_Calendar_NC.md](guides/Planting_Calendar_NC.md) — is it the right season?
2. Find it in [Master_Seed_and_Plant_List.md](inventory/Master_Seed_and_Plant_List.md) — where should it go?
3. Read that garden's plan in `beds/` — exactly where and how to plant it
4. Check [Planting_How_To.md](guides/Planting_How_To.md) — technique details

### "What's wrong with my [plant]?"
1. Check [Garden_Care_and_Troubleshooting.md](guides/Garden_Care_and_Troubleshooting.md) — symptom diagnosis
2. Find treatment/prevention steps
3. For herbs: check [Medicinal_Herb_Kit_Guide.md](guides/Medicinal_Herb_Kit_Guide.md) for uses

### "What do I do this week?"
1. Check [Garden_Todo_List.md](guides/Garden_Todo_List.md) by current season
2. Or check [March_Schedule.md](guides/March_Schedule.md) if it's March (template for other months)

### "I want to use herbs for [healing purpose]"
1. Check [Medicinal_Herb_Kit_Guide.md](guides/Medicinal_Herb_Kit_Guide.md)
2. Find which herbs treat that
3. Find them in [Master_Seed_and_Plant_List.md](inventory/Master_Seed_and_Plant_List.md) to confirm you own them

### "What grows in this garden?"
1. Read the garden file in `beds/`
2. Check the CSV grid if it exists (Bloomveil, Edgeworth)

### "I want native plants"
1. Check [NC_Native_Recommendations.md](reference/NC_Native_Recommendations.md)
2. Assign them in [Wayward_Meadow.md](beds/Wayward_Meadow.md) or [Edgeworth_Corner.md](beds/Edgeworth_Corner.md)

## 🔄 How This Repository Works

### Active Development Cycle
1. **Plan** — Decide what goes where using `beds/` guides
2. **Track** — Add to [Master_Seed_and_Plant_List.md](inventory/Master_Seed_and_Plant_List.md)
3. **Plant** — Follow [Planting_How_To.md](guides/Planting_How_To.md) and calendar
4. **Maintain** — Use [Garden_Care_and_Troubleshooting.md](guides/Garden_Care_and_Troubleshooting.md)
5. **Learn** — Update guides with what worked/didn't work
6. **Update** — Files get revised each year with new learnings

### Information is Organized By:
- **Location** — which garden (beds/)
- **Task** — what to do (guides/)
- **Asset** — what you own (inventory/)
- **Reference** — quick facts (reference/)
- **History** — why decisions were made (archive/)
